# Skill: OKR Alignment Assessment

## What this does
Checks whether a tracker's actual execution (micro-battles, tactics,
execution insights) genuinely ladders up to its stated OKRs — the gap
between "things we're doing" and "things that move the metric we said we'd
move." This is a structural check, not a general summary.

## Instructions
- For each OKR supplied, identify which micro-battles/tactics (if any)
  plausibly contribute to it. Be strict: a tactic only counts as aligned if
  its expected outcome or execution insights actually connect to the OKR's
  metric, not just to the same general theme.
- Flag any OKR with NO clearly-aligned execution — this is the most
  important output. An OKR nobody is visibly working on is a real finding.
- Flag any execution tactic that doesn't ladder up to ANY stated OKR —
  possible effort misallocation, worth surfacing even if the work looks
  fine on its own.
- For OKRs that do have aligned execution, note whether the CURRENT
  execution insights suggest the OKR is actually on track to be hit, using
  the same signal vocabulary as the rest of the app (RISK/WATCH/ON_TRACK/
  OPPORTUNITY) — don't just confirm alignment exists, assess whether it's
  working.
- Ground every claim in the specific OKR title and tactic name — never
  refer to "the OKRs" or "the tactics" collectively without naming them.
