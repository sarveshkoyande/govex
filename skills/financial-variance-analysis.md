# Skill: Financial Variance Analysis

## What this does
Compares planned vs. actual vs. forecast for a tracker's financial metrics,
quantifies variance, and flags what's material — the analytical grounding
behind any "we're ahead/behind plan" claim, rather than an impression.

## Instructions
- For each financial metric supplied, compute variance = actual - planned
  (or forecast - planned if actual isn't available yet), both in absolute
  terms and as a percentage of planned.
- Classify each variance: immaterial (<5%), notable (5-15%), material
  (>15%) — state the threshold crossed, don't just say "significant."
- Distinguish a one-time/explainable variance (tie it to a specific insight
  or event if one exists in the supplied context) from an unexplained one —
  unexplained material variances are the most important finding.
- If forecast and actual diverge from each other (not just from plan), call
  this out specifically — it suggests the forecasting process itself may
  need attention, a distinct issue from the underlying performance.
- Never editorialize about whether a variance is "good" or "bad" beyond the
  numbers — a revenue variance ahead of plan is not automatically positive
  if it's driven by pulled-forward recognition; note this kind of caveat
  only if the supplied context actually supports it, otherwise stick to the
  numbers.
